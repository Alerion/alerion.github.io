Важность документации

Обсудить важность документации, важность для open source проектов. Способы организации документации.

Способы организовать документацию

  • reStructuredText, Markdown, docbook
  • Wiki, Confluence
  • Google Docs
  • Комментарии в коде
Google Docs - удобно тем что групповое редактирование, применяются для рисования часто всяких диаграмм.
Что еще стоит упомянуть? LaTeX?

Требования к инструментам для документации

  • Текстовой формат хранения документа(для удовной работы в VCS).
  • Возможность оформления и стилизации.
  • Возможность сборки в различные форматы.
  • Перекрестные ссылки, DRY.

Документация Django

  • Считается большим плюсом Django.
  • Объемная. В PDF 1300+ страниц.
  • Постоянно развивается.
  • Много ньюансов описано за последние годы.

Как бы нам переводить документацию Django?

  • Она хорошая.
  • Ее постоянно поддерживают и обвноляют.
  • Sphinx уже делает всю сборку.
  • Но есть проблема с обновлением перевода.
Как у нас были идеи перевести докуменацию, но не было простых способов поддерживать перевод. Еще хотели перевести Pro Django.

Sphinx 1.1 (Oct 9, 2011)

Начало перевода Feb 15, 2012. Цель перевести основные разделы. Аналогичные проекты не ушли дальше двух глав.

Sphinx

Инструмент для создания документации на reStructuredText. C открытым исходным кодом. BDS лицензия.

reStructuredText

Язык разметки текстовых данных.

Autodoc

Генерация опиcания кода из docstrings.

			.. automodule:: io
			   :members:

Можно ссылать на описание функций из внешней документации.

			:py:func:`io.open`

sphinx-apidoc

Создает с нуля документацию API используя docstrings.

			$ sphinx-apidoc [options] -o outputdir packagedir [pathnames]
